PrologMCP：当 MCP 遇到 Prolog，Agent 推理有了形式化外挂
arXiv 2026GitHub
论文开源Agent推理
核心：提出把 SWI-Prolog 封装为 MCP Server，让 LLM Agent 通过标准接口执行符号推理，而非在文本中硬推复杂逻辑。
方法：LLM 负责语义翻译（事实/规则/查询→Prolog），Prolog 求解器执行形式化推理，MCP 负责连接，组成可执行、可检查、可复现的复合 Agent。
结果：在 PARARULE-Plus 多步演绎推理上，Prolog Formalizer 比 Standard LLM 与 Reasoning LLM 更稳定；推理过程可结构化诊断、可审计。
复现：✅ 论文与代码思路开源，SWI-Prolog 后端，可本地部署 MCP Server。

VeGAS：具身 Agent「先验证再行动」，动作选择更稳
arXiv 2026Embodied Agent
论文验证器具身智能
核心：提出 Verifier-Guided Action Selection（VeGAS），让具身 Agent 先采样一组候选动作，再由验证器筛选最优动作，而非直接输出单步动作。
方法：policy 采样多个候选动作，引入轻量验证器对动作后果打分，选择高分动作执行，将 test-time verification 引入具身决策闭环。
结果：在 embodied agent 基准上显著降低错误动作率，缓解「急于输出」导致的失败，多步任务成功率明显提升。
复现：✅ 论文 arXiv:2605.12620 已公开，方法与评估可复现（代码见 arXiv 页）。
